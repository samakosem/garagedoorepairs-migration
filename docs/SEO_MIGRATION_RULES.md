# SEO Migration Rules — garagedoorepairs.com

This is a critical SEO migration project for https://garagedoorepairs.com/.

The current site is WordPress. The new site will be built as a clean, high-performance code-based website, likely with Next.js.

## Main Goal

Build a premium, fast, unique, SEO-first website for Garage Door Repair Los Angeles without losing:

- Indexed pages
- Current rankings
- Backlinks
- Domain authority
- Existing SEO value
- Important URL structure

## Primary SEO Keyword

Garage Door Repair Los Angeles

## Secondary Keywords

- Emergency Garage Door Repair
- Garage Door Off Track Repair

## Critical Rule

Do not delete, rename, merge, or change any existing URL without checking:

1. Full URL inventory
2. Google Search Console Pages export
3. Ahrefs backlinks by target URL
4. Screaming Frog crawl
5. Redirect Map decision

Every URL must have one of these decisions:

- Keep
- Rebuild
- 301 Redirect
- Noindex
- Delete only if confirmed safe

## Technical SEO Requirements

The new site must include:

- Next.js App Router
- Server-rendered SEO pages
- Unique metadata per page
- Canonical tags
- XML sitemap
- robots.txt
- LocalBusiness schema
- Service schema
- FAQ schema where relevant
- Breadcrumb schema
- Internal links between services and locations
- Clean heading structure
- One clear H1 per page
- Fast Core Web Vitals
- Mobile-first design
- Sticky mobile call CTA
- Click-to-call buttons
- Noindex for thank-you, admin, test, staging, and private pages

## Design Direction

The design must be premium, unique, modern, trustworthy, local, and conversion-focused.

The website should feel like a serious Los Angeles garage door repair brand, not a generic contractor template.

Design priorities:

- Strong hero section
- Emergency repair CTA
- Trust signals
- Service cards
- Location/service sections
- Before/after style visuals if available
- Testimonials
- FAQ sections
- Professional typography
- Fast animations only if they do not hurt performance

## Forbidden

Do not:

- Change important URLs without 301 redirects
- Delete pages without SEO review
- Build a SPA-only site
- Remove SEO copy
- Replace SEO pages with short generic sections
- Use heavy animations that hurt speed
- Create fake locations
- Create duplicate thin city pages
- Forget sitemap, robots, canonicals, schema, or metadata
